Cygwinの世界観

Cygwinに関する世界観です。

GNU on Windows

Cygwin

Windowsで動くUNIX互換環境。ただしソフトウェアの動作にはCygwin1.dllが必要。

  • Cygwin - Wikipedia

ソースコード

  • git

MinGW

GNUのC/C++コンパイラをWindowsに対応させたもの。僕としては、EclipseのCDT（C/C++開発ツール）をインストールするのがお勧めです。

Windows APIのためのヘッダファイルを含んでいるため、Windowsアプリケーションの開発のために使える、と言うことらしいが、本当にきちんと動くかは怪しいところ。

Eclipseも参照のこと。

  • MinGW - Wikipedia

Cygwin/X

Cygwin/XはCygwin環境上で動くX11。X11設定とプログラミングを参照のこと。

コンパイラとして

GCC/MinGW + MSYS

GCCはGNUによるコンパイラ。MinGWと言うWindows版もある。

MSYSはWindows上で動作するBashやUNIXユーティリティ。MinGWを補う目的で作られた。

GNUツールチェインも参照のこと。

Cygwin

MinGWを使うのも良いが、WindowsでUNIX環境を構築できるCygwinを使うという手もある。この場合、ランタイムライブラリのCygwin1.dllを開発したプログラムに同梱する必要がある。

簡単な説明

互換レイヤー

  • その他のOS
    • 互換レイヤー
      • Cygwin
        • WindowsでUNIXコマンドを使いたいならCygwinを入れよう
        • WindowsでGCCを使いたいのであれば、MinGW, MSYSを使うことでフリーなC/C++のコンパイル環境が整う
        • LLVM/ClangをWindowsで使うという選択肢もあり
      • Wine
        • WindowsアプリケーションをLinuxなどで動かすための互換レイヤー
        • 実際は全てのアプリケーションが完全に動作するわけではない
      • WSL
        • WSL1はWindowsでLinuxアプリケーションがネイティブ動作する互換レイヤーだった
        • WSL2はWindows上でLinuxカーネルを動作させる仮想環境になった
      • コマンドプロンプト、PowerShell
        • Windowsでコマンド操作を行う時は、コマンドプロンプトを使う
        • 最近ではPowerShellと呼ばれる従来よりも高機能なシェルがあり、仮想環境の構築などの時にも使う
