Xfce

Xfceとは

見た目のかっこよさやカスタマイズ性は十分にあるままで、軽量かつ高速なUNIXのデスクトップ環境を作ろう、という試み。

GNOMEやKDEの巨大指向にうんざりした人にお勧めで、一部では「絶対にこの方が（UNIXとして）正しい」というファンが居るデスクトップ環境。リーナス・トーバルズも一時期GNOMEからXfceに乗り換えていた。

ある意味、Linuxのデスクトップ環境は、肥大化・巨大化してWindowsよりも不安定になっている。本当に路線として正しいのは、「UNIXの哲学を体現する」Xfceなのかもしれない。一部では商用UNIXで用いられるCDEに似ているとか。

Thunarという簡素で賢いファイルマネージャも人気。Xfce-Goodiesという軽量アプリケーションのプロジェクトもあり、軽量なWebブラウザのMidoriなどが開発されている。

標準の機能は少ないが、プラグインを導入することで機能を拡張できるため、コアなユーザにも推奨できる。見た目のスタイルを変えられるテーマ機能もある。

最近、Xfceユーザーになった

最近、僕はGNOME 3を使うのをやめ、Xfceユーザーになりました。

Debianなら、aptコマンドですぐに導入できます。

# apt install xfce4 xfce4-goodies

ただし、GNOMEとの混在環境ではアプリケーションメニューなどが分かりづらくなるため、最初のDebianのインストール時にXfceを選択することをおすすめします。

今はDebianでXfceを使っています。軽くて、安定していて、そしてGNOMEよりもどこに機能があるのか分かりやすく、とても快適です。

WindowsやMacのユーザーに、「Linuxデスクトップはこんなに使いやすいのだ」と訴えかけるには、ディストリビューションのデフォルトの環境は、GNOME 3よりもXfceにするべきだと思います。

Xfceを使うことで、GNOME 3が全く「本当のLinux環境ではない」ということを思い知らされました。

ただし、今のところGNOME 3のアプリケーションとXfceのアプリケーションがメニューに混在していて使いづらいため、一度XfceをベースとしたDebianあるいはほかのディストリビューションを導入しなおそうと思います。

XfceでMozilla Firefoxを使うと、WindowsとFirefoxアカウントの情報を共有できるため、とても便利です。また、GIMPやInkscapeなどはWindows版のものよりも高速に起動します。

ただし、教訓として、最初にGNOMEを導入したのがまずかったように、Linuxは何度も入れなおす必要があるということが分かります。そのため、大事なデータはWindowsに残し、USBハードディスクにバックアップを取るようにしています。

GNOMEデスクトップも参照のこと。

Xfceを入れる前によく考えよう

軽い気持ちでXfceをaptなどからインストールする前に、本当にXfceがシステムに必要なのかをよく考えましょう。

Xfceをaptなどからインストールすると、依存関係などからさまざまなパッケージが大量にインストールされてしまい、後でアンインストールすることが困難になります。

また、標準的なGNOMEなどに比べて、標準的でないXfceなどのデスクトップ環境は、バグや不具合が多く、きちんと動かない（つまり使い物にならない）ことが多々あります。

このような時、できることは、ディストリビューションを再インストールしてシステムの再設定を行うか、不要であってもXfceとGNOMEなどが混在した環境でGNOMEを使うことです。

ですが、GNOMEとXfceが共存したところで、ほとんど良いことは何もありません。GNOMEだけを使うので十分です。FirefoxなどはGNOMEだけの環境ではGNOMEのファイルマネージャなどを標準で連携などの際に使いますが、Xfceが混在していると、使わないにもかかわらずGNOMEではなくXfceのファイルマネージャが使われたりして、システムが非常に使いづらくなってしまいます。

また、そもそも、Xfceを追加でインストールすること自体、僕はおすすめしません。もし、Xfceを本当に使うならば、ディストリビューションを最初にインストールする際にXfceを選択するか、Xfceを標準のデスクトップ環境に採用したディストリビューションをインストールしたほうがいいでしょう。

なので、Xfceをインストールする前に、使うのか使わないのかをよく考えましょう。単に「使ってみたいから使う」という考え方は推奨しません。
